POO

La programmation procédurale:

Consiste à écrire des procédures ou des fonctions qui effectuent des opérations sur les données.

La POO:

consiste à créer des objets contenant à la fois des données et des fonctions. Exple: une class contenant des fn et des données

Avantages de la POO:

plus rapide et plus facile à exécuter, fournit une structure claire pour les programmes, empeche la repetition ds le code..
La POO permet de créer des applications entièrement réutilisables avec moins de code et un temps de développement plus court, por ce fait on doit extraire les codes communs à l'application, les placer au même endroit et les réutiliser au lieu de les répéter.

Les class et les objet

Les classes et les objets sont les deux principaux aspects de la programmation orientée objet.

Les class

Une classe est un modèle pour les objets. Une classe est définie à l'aide du class mot-clé, suivi du nom de la classe et d'une paire d'accolades ({}). Toutes ses propriétés et méthodes se trouvent à l’intérieur des accolades:
class Fruit {
// Properties
public $name;
public $color;
// Methods
function set_name($name) {
$this->name = $name;
}
function get_name() {
return $this->name;
}
}

Les objets

Un objet est une instance de classe. Nous pouvons créer plusieurs objets à partir d'une classe. Chaque objet possède toutes les propriétés et méthodes définies dans la classe, mais elles auront des valeurs de propriété différentes.
Les objets d'une classe sont créés à l'aide du mot-clé new.
Dans l'exemple ci-dessous, $apple et $banana sont des instances(objets) de la classe Fruit :
class Fruit {
// Properties
public $name;
public $color;
// Methods
function set_name($name) {
$this->name = $name;
}
function get_name() {
return $this->name;
}
}
// Les objets
$apple = new Fruit();
$banana = new Fruit();
$apple->set_name('Apple');
$banana->set_name('Banana');
echo $apple->get_name();

"; echo $banana->get_name();
